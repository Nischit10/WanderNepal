# User Story 11: GET /api/destinations/:id - Single Destination Detail

## Description
As a user, I want to see destination details so I can decide to book.

## Endpoint
`GET /api/destinations/:id/`

## Response
### Success (200 OK)
Returns a single destination object.
```json
{
    "id": 1,
    "name": "Pokhara",
    "city": "Pokhara",
    "country": "Nepal",
    "description": "Beautiful lake city",
    "image_url": "http://example.com/image.jpg",
    "price_per_night": "50.00",
    "category": "Nature",
    "rating": 4.5
}
```

### Not Found (404 Not Found)
Returns if the destination with the given ID does not exist.
```json
{
    "detail": "Not found."
}
```

## Acceptance Criteria
- [x] Returns 200 + object for valid ID.
- [x] Returns 404 for invalid ID.
- [x] Tested with automated tests in `backend/api/tests.py`.

## Implementation Details
- Added `DestinationDetailView` to `backend/api/views/destination.py`.
- Registered route `destinations/<int:pk>/` in `backend/api/urls.py`.
- Exported `DestinationDetailView` in `backend/api/views/__init__.py`.
