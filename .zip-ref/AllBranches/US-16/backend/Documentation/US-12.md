# US-12: Create New Booking

## Summary
The goal of this task was to implement an endpoint that allows logged-in users to create a booking for a specific destination. The system calculates the total price based on the duration of the stay and the destination's price per night.

## Actions Taken

### 1. Model Creation
Created the `Booking` model in `backend/api/models/booking.py` with the following fields:
- `user` (ForeignKey to User)
- `destination` (ForeignKey to Destination)
- `start_date` (DateField)
- `end_date` (DateField)
- `total_price` (DecimalField)
- `created_at` (DateTimeField)

Updated `backend/api/models/__init__.py` to include the `Booking` model.

### 2. Serializer Implementation
Created `BookingSerializer` in `backend/api/serializers/booking.py`.
- Maps `destinationId`, `startDate`, and `endDate` from the request body to model fields.
- Validates that `startDate` is in the future and `endDate` is after `startDate`.
- Automatically calculates `totalPrice` as `pricePerNight * nights`.

### 3. View Implementation
Created `BookingCreateView` in `backend/api/views/booking.py`.
- Endpoint: `POST /api/bookings/`
- Requires `TokenAuthentication`.
- Requires `IsAuthenticated` permission.
- Saves the booking with the currently logged-in user.

### 4. URL Configuration
Registered the new endpoint in `backend/api/urls.py`:
- `path('bookings/', BookingCreateView.as_view(), name='bookings')`

### 5. Database Migration
Generated and applied migrations:
- `python manage.py makemigrations api`
- `python manage.py migrate`

## Endpoint Details

### POST /api/bookings/
**Headers:**
- `Authorization: Token <your_token>`

**Body:**
```json
{
  "destinationId": 1,
  "startDate": "2026-06-01",
  "endDate": "2026-06-05"
}
```

**Response (201 Created):**
```json
{
  "id": 1,
  "destinationId": 1,
  "startDate": "2026-06-01",
  "endDate": "2026-06-05",
  "totalPrice": "200.00",
  "created_at": "2026-05-08T..."
}
```

## How to Run and Test

### 1. Start the Local Server
From the root directory of the project, activate your virtual environment and run the server:
```powershell
# Activate venv (Windows)
.\venv\Scripts\activate

# Run migrations (if not already done)
python backend/manage.py migrate

# Start the server
python backend/manage.py runserver
```
The server will be available at `http://127.0.0.1:8000/`.

### 2. Testing with Postman

Since this endpoint requires authentication, you must first log in to get a **Token**.

#### Step A: Get your Token (Sign In)
- **Method:** `POST`
- **URL:** `http://127.0.0.1:8000/api/signin/`
- **Body (JSON):**
  ```json
  {
    "email": "your_email@example.com",
    "password": "your_password"
  }
  ```
- **Response:** Copy the `"token"` value (e.g., `9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b`).

#### Step B: Create a Booking
- **Method:** `POST`
- **URL:** `http://127.0.0.1:8000/api/bookings/`
- **Headers:** 
  - Key: `Authorization`
  - Value: `Token 9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b` (Note the word **Token** followed by a space)
  - Key: `Content-Type`, Value: `application/json`
- **Body (JSON):**
  ```json
  {
    "destinationId": 1,
    "startDate": "2026-06-10",
    "endDate": "2026-06-15"
  }
  ```

### What is a Token?
A **Token** is like a temporary ID card. Once you log in (Step A), the server gives you this "card". You must show it (Step B) in the `Authorization` header so the server knows who you are without you having to send your password every time.
