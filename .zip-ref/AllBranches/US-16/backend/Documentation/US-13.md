# US-13: List Logged-in User's Bookings

## Goal
As a logged-in user, I want to see all my bookings.

## Endpoint
`GET /api/bookings/me`

## Implementation Details

### Model
- Uses the existing `Booking` model which has a foreign key to `User` and `Destination`.

### Serializer
- Updated `BookingSerializer` in `backend/api/serializers/booking.py` to include `destinationName`.
- `destinationName` is a read-only field that sources its value from `destination.name`.

### View
- Implemented `UserBookingListView` in `backend/api/views/booking.py`.
- Requires authentication (`IsAuthenticated`).
- Filters `Booking` objects by the current `request.user`.
- Sorts bookings by `created_at` descending (newest first).

### URL Configuration
- Registered the endpoint in `backend/api/urls.py` as `path('bookings/me/', UserBookingListView.as_view(), name='my-bookings')`.

## Security
- Requires Token Authentication.
- Returns 401 Unauthorized if no valid token is provided.
- Only returns bookings belonging to the authenticated user.

## Testing
- Added `test_get_my_bookings` to `backend/api/tests_booking.py` to verify:
    - Successful retrieval of user's bookings.
    - Presence of `destinationName` in the response.
    - Correct sorting (descending by creation date).
- Added `test_get_my_bookings_unauthorized` to verify 401 response for unauthenticated users.
