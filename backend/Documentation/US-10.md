# US-10: Implement `GET /api/destinations` endpoint

## Summary
The goal of this task was to create an API endpoint that allows users to browse all available destinations. This endpoint (`GET /api/destinations`) returns an array of serialized destination objects with a 200 OK status code.

## Actions Taken

### 1. Serializer Creation
Created the `DestinationSerializer` in `backend/api/serializers/destination.py` using Django REST Framework's `ModelSerializer`. This automatically handles the conversion of `Destination` model instances into a JSON format.
Updated `backend/api/serializers/__init__.py` to expose the new serializer.

### 2. View Implementation
Created the `DestinationListView` in `backend/api/views/destination.py`. Utilized DRF's `generics.ListAPIView`, which connects the `Destination` model and the `DestinationSerializer` to automatically handle fetching all records from the database and returning them as an array.
Updated `backend/api/views/__init__.py` to expose the new view.

### 3. URL Routing
Added the `destinations` route in `backend/api/urls.py` and mapped it to the `DestinationListView`. This makes the endpoint accessible at `/api/destinations`.

## Verification
- Verified that `python manage.py check` runs without any issues.
- A `GET` request to `/api/destinations` (e.g., via Postman) successfully returns a `200 OK` status with an array of destination objects (or an empty array if no data is present).
