# US-17: Standardize all API error response format

## User Story
As a developer, all APIs must return consistent JSON error responses so the frontend can handle errors reliably.

## Format
All error responses (400, 401, 403, 404, 500) now return the following JSON structure:
```json
{
    "success": false,
    "error": "ERROR_TYPE",
    "message": "Detailed error message here"
}
```

## Changes Made
1.  **Created `backend/api/exceptions.py`**:
    *   Implemented `custom_exception_handler` to catch all DRF exceptions.
    *   Added logic to format the response into the required `{ success, error, message }` structure.
    *   Categorized errors into `VALIDATION_ERROR` (400), `AUTHENTICATION_ERROR` (401), `PERMISSION_DENIED` (403), `NOT_FOUND` (404), and `SERVER_ERROR` (500).
2.  **Modified `backend/core/settings/base.py`**:
    *   Added `REST_FRAMEWORK` configuration to use the `custom_exception_handler`.
3.  **Updated `backend/api/tests.py`**:
    *   Updated `test_get_destination_detail_invalid` to verify the new error format.

## What to Expect
*   Any API error will now have a consistent structure.
*   Frontend developers can rely on `response.data.success` to check for errors and `response.data.message` to display to the user.
*   Example 404 Response:
    ```json
    {
        "success": false,
        "error": "NOT_FOUND",
        "message": "Not found."
    }
    ```
*   Example 400 Response (Validation):
    ```json
    {
        "success": false,
        "error": "VALIDATION_ERROR",
        "message": "rating: Ensure this value is less than or equal to 5.0."
    }
    ```
