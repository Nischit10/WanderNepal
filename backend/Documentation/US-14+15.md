# US-14+15: Booking Detail and Cancellation

## Goal
- **US-14:** As a logged-in user, I want to view the details of a specific booking.
- **US-15:** As a logged-in user, I want to cancel one of my bookings.

## Implementation Details

### Model Updates
- Added a `status` field to the `Booking` model in `backend/api/models/booking.py`.
- Status choices: `PENDING`, `CONFIRMED`, `CANCELLED`.
- Default value is `CONFIRMED`.

### Serializer Updates
- Updated `BookingSerializer` in `backend/api/serializers/booking.py` to include the `status` field.
- The `status` field is read-only for general booking operations.

### View Updates
- Implemented `BookingDetailView` in `backend/api/views/booking.py`.
    - `GET /api/bookings/:id/` returns full booking details.
    - Checks for user ownership (returns 403 Forbidden if the booking belongs to another user).
    - Returns 404 Not Found if the booking does not exist.
- Implemented `BookingCancelView` in `backend/api/views/booking.py`.
    - `PATCH /api/bookings/:id/cancel/` sets the status to `CANCELLED`.
    - Checks for user ownership (returns 403 Forbidden if the booking belongs to another user).
    - Returns 404 Not Found if the booking does not exist.
    - Returns the updated booking object on success.

### URL Configuration
- Registered the new endpoints in `backend/api/urls.py`:
    - `path('bookings/<int:pk>/', BookingDetailView.as_view(), name='booking-detail')`
    - `path('bookings/<int:pk>/cancel/', BookingCancelView.as_view(), name='booking-cancel')`

## Security
- Both endpoints require Token Authentication (`IsAuthenticated`).
- Strict ownership check ensures users can only view or cancel their own bookings.

## Testing
- Added comprehensive tests in `backend/api/tests_booking.py`:
    - `test_get_booking_detail_success`: Verifies successful retrieval of booking details.
    - `test_get_booking_detail_forbidden`: Verifies 403 response when accessing another user's booking.
    - `test_cancel_booking_success`: Verifies successful cancellation and status update.
    - `test_cancel_booking_forbidden`: Verifies 403 response when trying to cancel another user's booking.
