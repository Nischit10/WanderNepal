# US-09: Seed database with 10+ destinations

## Summary
The goal of this task was to provide the application with realistic seed data for testing purposes. This involved creating a `Destination` model and a mechanism to populate it with at least 10 entries.

## Actions Taken

### 1. Model Creation
Created the `Destination` model in `backend/api/models/destination.py` with the following fields:
- `name` (CharField)
- `city` (CharField)
- `country` (CharField)
- `description` (TextField)
- `image_url` (URLField)
- `price_per_night` (DecimalField)
- `category` (CharField)
- `rating` (FloatField)

Updated `backend/api/models/__init__.py` to expose the new model.

### 2. Database Migration
Generated and applied a new migration to create the `destinations` table in the database:
- `python manage.py makemigrations`
- `python manage.py migrate`

### 3. Seed Script Implementation
Created a Django management command `seed_destinations` in `backend/api/management/commands/seed_destinations.py`. This script populates the database with 11 realistic destinations across Nepal, covering various categories like Nature, Culture, Adventure, Wildlife, and History.

### 4. Data Seeding
Executed the seed command:
`python manage.py seed_destinations`

Output:
- Successfully seeded Phewa Lake
- Successfully seeded Boudhanath Stupa
- ...
- Successfully seeded all destinations

### 5. Admin Registration
Registered the `Destination` model in `backend/api/admin.py` to allow for easy viewing and management via the Django Admin interface.

## Verification
- Run `python manage.py seed_destinations` again to ensure it handles updates gracefully (idempotent).
- Check the `api_destination` table in the database to confirm 11 rows exist.
- Access the Django Admin panel to view the seeded data.
